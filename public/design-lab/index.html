Pillar vs supporting density

Best-practice component counts by post type. Rule of thumb: components should feel earned, not routine.

Component Pillar (3K+ words) Supporting (800–1,500 words)
Stat grid1 in the hero area0–1 (only if numbers drive the piece)
Callouts3–5 spread through the body1–2 max
Pull quotes2–3 at section transitions0–1 (only if there's a genuinely quotable line)
Comparison table1 recommendedOptional; only if it's a listicle/comparison piece
Hero imageRequiredRequired (cluster-themed generative treatment is fine)

The operations-cost-mistake sample is 5 elements on 1,000 words — that's on the heavy side, intentional so you could see the full component range at once. For the batch pass on the other 12 sub-posts I'd propose: 1 stat grid (only if the post is number-driven), 1–2 callouts, and 1 pull quote maximum. That's 2–4 elements per post, which respects brevity.

Ratio rule of thumb: roughly 1 editorial break element per 500–700 words of supporting content, and per 800–1,000 words of pillar content. Pillars can breathe; sub-posts need to respect the reader's time.

One hard rule: never stack two break elements back-to-back (callout immediately after a stat grid, pull quote immediately after a callout). There must be prose between each one, otherwise the page starts to feel like a slide deck instead of an article.
